Some overview

Learning HTML is really, really easy. All you really need to know is the gist of what syntax to use, and VS code (or any other smart IDEs,) will autocomplete the rest of the syntax for you! For example, if I want a paragraph, I just utilize the tag that structure web content, and let's say I want a paragraph. Then, I can just add "< >!" The most important thing that'll get you well with HTML is knowing the skeleton and structure of html, and how it interacts with CSS and JavaScript.


Let's expand the acronym of what we're going to be working on.

  • HTML = Hypertext Markup Language
  • CSS = Cascading Style Sheets
  • JS = JavaScript
  • An extra link on getting started with your first website.

    • Why these acronyms? I don't know. You can check out the hyperlinks above, the Mozilla Foundation provides great documentation.

    Before I go over headings, I'd like to explain about HTML documents and their basic structures. Every HTML's basic structure consists of HTML, CSS, and JS. It is put into a folder and read by the browser. Here's a short, concise explanation of the three different files:

    1. index.html
      2. This is the the instructions on how to manage the CSS and JS. The HTML resides here.
    2. style.css
      3. This file is the way you would style your HTML to make it all pretty and dandy.
    3. script.js
      4. This explains the mechanics behind the features of the UI, the 'brains' of the website (not technically speaking!) Without this, this website would just be a one big fat decoration.

    Attributes

    Tags can have oops attributes that provide extra information:

    • (class="my-class" for styling multiple elements)
    • (id="unique-id" for one specific elements)
    • (style="color: red;" for inline CSS)
    • (href, src, alt as shown above)

    Semantic HTML

    Modern HTML uses meaningful tags:

    • (<header>, <nav>, <main>, <footer> for page structure)
    • (<article>, <section> for content areas)
    • (<button> for clickable buttons)
    • (<form>, <input>, <label> for user input)
